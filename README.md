# Video Monitor Hub

Video Monitor Hub is a web application for registering security cameras, viewing recent motion recordings, and managing saved video updates. The project combines a Laravel web application, a PostgreSQL database, and a Python/OpenCV processor for motion detection and video recording.

## Main Features

- User registration, login, and logout.
- Camera registration with name, access URL, and local file storage path.
- Camera listing with view, edit, and delete actions.
- Recent camera updates page with recorded video logs.
- Modal video playback for recordings generated by motion detection.
- Python processor that reads registered cameras, detects motion from camera streams, records video files, and stores update logs.

## Running With Docker

Start the application stack:

```bash
docker compose up -d --build
```

On first startup, the Laravel container prepares the application runtime by:

- copying `laravel-app/.env.example` to `laravel-app/.env` when `.env` does not exist;
- applying Docker database defaults to that new `.env`;
- installing Composer dependencies into a Docker volume;
- generating `APP_KEY` when needed;
- creating Laravel's `public/storage` link when needed;
- installing Node dependencies into a Docker volume;
- building the Vite frontend assets when needed.

The Laravel application is exposed at:

```text
http://localhost:8000
```

PostgreSQL is exposed on port `5432`. The default Docker database values are:

```text
DB_DATABASE=video_monitor
DB_USERNAME=postgres
DB_PASSWORD=secret
```

Inside `laravel-app/.env`, configure the Laravel database connection for Docker:

```text
DB_CONNECTION=pgsql
DB_HOST=database
DB_PORT=5432
DB_DATABASE=video_monitor
DB_USERNAME=postgres
DB_PASSWORD=secret
```

Then run the Laravel setup commands inside the application container:

```bash
docker compose exec laravel-app php artisan migrate
```

## Debugging With Xdebug

The Laravel Docker image includes Xdebug for local step debugging in VS Code.

1. Install the recommended VS Code extensions from `.vscode/extensions.json`.
2. Open the Run and Debug panel.
3. Select `Listen for Xdebug`.
4. Start debugging.
5. Add a breakpoint in a Laravel controller, route, request, or model.
6. Use the application at `http://localhost:8000`.

Xdebug listens on port `9003` and maps the container path `/var/www/html` to the local `laravel-app` folder.

To disable Xdebug temporarily:

```bash
XDEBUG_MODE=off docker compose up -d --build laravel-app
```

## Python Motion Processor

The Python processor connects to Laravel through internal API endpoints. Docker gives both containers the same `PROCESSOR_API_TOKEN`, so the processor can:

- request active cameras from `GET /api/processor/cameras`;
- process each camera stream with OpenCV;
- write finished clips to the shared recordings volume;
- register each saved recording through `POST /api/processor/videos`.

The shared recordings volume is mounted at:

```text
python-processor: /app/storage/videos
laravel-app:      /var/www/html/storage/app/public/videos
```

Laravel serves those files through the standard public storage URL:

```text
/storage/videos/{filename}
```

The Python processor can still run without a configured stream. In that mode it stays alive and waits for active cameras from Laravel.

To test one camera stream manually, provide `PROCESSOR_CAMERA_URL` when starting the stack:

```bash
PROCESSOR_CAMERA_URL=http://example-camera.local/video docker compose up -d --build python-processor
```

Recorded motion clips are written inside the Python container to:

```text
/app/storage/videos
```

To reset the database during development, remove the database volume and run migrations again:

```bash
docker compose down -v
docker compose up -d --build
docker compose exec laravel-app php artisan migrate
```
